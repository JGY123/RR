A6
"Overall" rank table — quintile groupings are mis-bucketed
High
Open
What we need
In the rank tables, holdings should be bucketed by their quintile rank. Q1 should contain only stocks with rank=1, Q2 only rank=2, etc. The sub-rank tables (REV, VAL, QUAL) do this correctly — Q1 weighted-avg comes out exactly 1.00, Q2 exactly 2.00, and so on. Will those anchor strings remain bit-identical across:

We've already had one parser-drift incident (April 2026) where 4 different parser code paths each interpreted column 15 differently after a format change, producing wacky on-screen numbers (like 60% idiosyncratic risk where the truth was 49%). B7
[Cash], @NA, [Unassigned] — what do these special row labels mean?
Confirmation
Question
Confirm:
• [Cash] = portfolio's cash allocation (assumed)
• @NA = holdings that couldn't be classified into a grouping dimension (sector, country)?
• [Unassigned] = same as @NA, or different concept?

Confirmation
